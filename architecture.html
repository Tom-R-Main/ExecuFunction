Architecture & Trade‑offs

Azure‑first, multicloud‑legible. Security‑first endpoints and budget discipline.

Topology

SWA (global CDN) → Azure Functions → (optional) Azure Table Storage.

  • Static Web Apps ≈ Firebase Hosting
  • Azure Functions ≈ Cloud Functions
  • (Optional) Azure Container Apps ≈ Cloud Run

Security stance

  • POST for auth starts (when added), persistent server‑side state
  • Cache‑control: no‑store on all /api/* responses
  • Owner‑only access patterns for tasks

Cost envelope

  • Free SWA tier for marketing site
  • Functions scale‑to‑zero
  • Context payload capped to ~1 KB
