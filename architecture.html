How it works

Executive Function runs on Azure Static Web Apps with small Azure Functions behind it. We keep data minimal (email for waitlist; short entries for rituals and tasks). Endpoints that touch personal info are marked no-store so they aren't cached by browsers or proxies. Certificates and HTTPS are managed automatically.

Hosting & APIs

SWA (global CDN) → Azure Functions → Azure Table Storage

  • Static site with automatic HTTPS and global edge delivery
  • Serverless functions that scale to zero when idle
  • Minimal data storage for waitlist, rituals, and tasks

Privacy & Safety

  • POST for auth flows (when added), persistent server-side state
  • Cache-control: no-store on all /api/* responses
  • Owner-only access patterns for personal data
  • Minimal data collection, deletable on request

Lightweight & scalable

  • Free SWA tier handles thousands of users
  • Functions scale to zero (pay only when used)
  • Tiny payloads (~1KB) keep costs minimal
  • No heavy frameworks or unnecessary complexity

Roadmap

  • ADHD first — calendar orchestration, next-step tasks, and two-minute ritual check-ins
  • Next cohorts — adapting for brain injury, autism, and addiction (with clinician input)
  • Agentic loops — safer, time-boxed "do this for me" flows that propose the next step
For engineers: multicloud mapping
  • Azure Static Web Apps ≈ Firebase Hosting / AWS Amplify
  • Azure Functions ≈ Cloud Functions / Lambda
  • Azure Table Storage ≈ Firestore / DynamoDB
  • Azure Container Apps ≈ Cloud Run / App Runner
